## はじめに
HTTPを学び始めた際、以下の疑問を持ちました

- HTTPリクエスト・レスポンスって、結局なに？
- HTTPメッセージはどんな形をしているの？

この記事では、HTTP/1.1を主軸として、リクエスト・レスポンスの基本構造を整理したいと思います。

## 結論

理解のポイントは次の2つです。

- HTTPリクエスト・レスポンスは、クライアントとサーバーがHTTPというルールに従ってメッセージをやり取りすること
- HTTP/1.1では、メッセージは「開始行 + ヘッダー + ボディ」で表現される

## 本題
### HTTPリクエスト・レスポンスとは

HTTPリクエスト・レスポンスとは、クライアントがサーバーへ要求を送り、サーバーが結果を返す仕組みです。

例えば、ブラウザでURLを開くと次の流れになります。

1. ブラウザがHTTPリクエストを送る
2. サーバーがHTTPレスポンスを返す
3. ブラウザがレスポンス内容を表示する

### HTTPとは
一言で言うと「WebブラウザとWebサーバー間の通信ルール(プロトコル)」です。

HTTPにはどのようなルールが決められているのか、簡単にまとめてみました。
| ルール | ルール内容 |
| ---- | ---- |
| ステートレス | 全てのリクエストはそれぞれ独立している |
| URL | URLを使用して、リソースの操作を行う |
| メソッド | メソッドを使用して、処理の目的を表現する |
| ステータスコード | 処理の結果をステータスコードとして返す |
| ヘッダー | ヘッダーを使用して、柔軟に機能を拡張できる |

補足として、HTTPで「何をしたいか」は主にメソッド(GET/POSTなど)で表し、「どの対象か」はURIで表します。

### プロトコルとは
一言で言うと「通信の動作をルールとして定めたもの」です。
では、そのルールはどのような種類があるのか簡単にまとめてみました。
| プロトコル名 | 役割 |
| ---- | ---- |
| FTP | ファイルのやり取り |
| DNS | ドメイン名からIPアドレスを探す |
| SSH | 外部のサーバーに暗号化してアクセス |
| TLS/SSL | セキュアな通信やり取り |
| IMAP | メールサーバーでメールを管理 |

### リクエスト・レスポンスはどんな構造をしているのか
ここでは、HTTP/1.1の基本形を紹介します。
HTTP/2の擬似ヘッダーは最後に補足します。
※HTTP1.1とHTTP/2の違いについては後述します

### リクエストの構造(HTTP/1.1)

例として、次のURLにGETリクエストを送るとします。
```
https://httpbin.org/get
```

URLは次のように読み取れます。

- `https`: スキーム(通信方式)
- `httpbin.org`: 接続先ホスト
- `/get`: 対象パス

HTTP/1.1のリクエストメッセージは、次のように「開始行 + ヘッダー + 空行 + ボディ」で構成されます。

```text
GET /get HTTP/1.1
Host: httpbin.org
Accept: application/json
User-Agent: Mozilla/5.0 (...省略...)
```

- 1行目: リクエスト開始行(メソッド/パス/HTTPバージョン)
- 2行目以降: リクエストヘッダー
- ヘッダーの後の空行: ヘッダー終端
- 空行の後: 必要ならリクエストボディ(POST/PUTなどで使用)

GETではボディなしの場合が一般的です。

### レスポンスの構造(HTTP/1.1)

同じリクエストに対するHTTP/1.1レスポンスの基本形は次のとおりです。

```text
HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 123

{"args":{},"headers":{"Host":"httpbin.org"},"url":"https://httpbin.org/get"}
```


- 1行目: ステータス行(HTTPバージョン/ステータスコード/理由句)
- 2行目以降: レスポンスヘッダー
- 空行の後: レスポンスボディ

### HTTP/2との違い

いざ、GoogleChromの開発者ツールでリクエストメッセージを確認したところ、次のような構造になっていました。

```text
:method: GET
:scheme: https
:authority: httpbin.org
:path: /get
```

プロパティの最初に`:`がついています。
これは一体なんでしょうか?

これはHTTP/2での表現で、HTTP/1.1の開始行に相当する情報です。
HTTP/1.1とHTTP/2では表現形式が異なりますが、本記事ではその違いの詳細には立ち入りません。
まずは、リクエストメッセージから「何を」「どの対象に」したいのかを読み取れるところまでをここでのゴールにしたいと思います。

## まとめ

- HTTPリクエスト・レスポンスは、HTTPというルールに基づいて要求と応答の1往復の仕組み
- HTTP/1.1では、メッセージは「開始行 + ヘッダー + ボディ」で理解できる
- HTTP/2は同じ意味論を持ちつつ、表現形式が異なる(擬似ヘッダーなど)
